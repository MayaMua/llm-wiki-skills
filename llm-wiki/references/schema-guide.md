# Schema Guide — Configuration & Frontmatter

Defines the `CLAUDE.md` config + schema document, the YAML frontmatter for every
page type, and the hierarchical index layout.

`CLAUDE.md` is the **single** config + schema file at the wiki root — there is no
separate `wiki-config.md`. Scope/domains, language, naming conventions, and private
zones live in `CLAUDE.md` (read at session start). The remaining machine settings are
**built-in script defaults**, not per-vault config:

| Setting | Default | Lives in |
|---------|---------|----------|
| Staleness (days) | high 90 / medium 180 / low 365 | `lint_wiki.py` |
| Article max words | 1200 | `references/article-guide.md` |
| Search tool | none (index scanning); qmd is opt-in | `query` Path A |
| Confidence gating | medium ≥3 sources, high candidate ≥5 | `SKILL.md` § Confidence gating |

Domains are read from `CLAUDE.md`'s `## Scope` → "What this wiki covers:" bullets.

## CLAUDE.md — The single config + schema document

`CLAUDE.md` sits at the wiki root. It tells the LLM the wiki's scope, naming
conventions, current state, and open questions.

**Co-evolve it with the wiki** — update after every major compile or ingest batch.

### Template

```markdown
# <Topic Title> Knowledge Base

> Schema document — read at the start of every session together with wiki/index.md.

## Scope
What this wiki covers:
- <bullet list>

What this wiki deliberately excludes:
- <bullet list>

## Operations
This wiki follows the llm-wiki skill's eight operations:
`scaffold`, `ingest`, `batch_ingest`, `query`, `lint`, `audit`, `reflect`, `compile`.

## Naming conventions
- **Source pages** (`wiki/sources/`): kebab-case slug
- **Concept pages** (`wiki/concepts/`): kebab-case slug
- **Entity pages** (`wiki/entities/`): kebab-case slug
- **Synthesis pages** (`wiki/synthesis/`): kebab-case slug

### Wikilinks & bilingual naming
- Wikilinks/slugs: always `[[category/english-lowercase-slug]]` — never the non-English name
- Bilingual vault: `title:` = the **primary-language** name; the page's first line / `# H1`
  leads with the primary language and annotates the secondary in `（）`. The scaffold writes
  a `### Bilingual format` block here naming which language leads, with an example
  (Chinese-primary → `经济（Economy）`; English-primary → `Economy（经济）`)
- `aliases`: **all** names, every language — so alignment/search match either

### Diagrams and formulas
- All diagrams are **mermaid**. No ASCII art.
- All formulas are **KaTeX**.

## Index note
wiki/index.md is auto-generated by `scripts/build_index.py`. Do not edit manually.
Domain catalogs live in wiki/index/<domain>.md.

## Open research questions
- <questions>

## Research gaps
- [ ] <URL> — why it's relevant

## Audit backlog
*(none)*

## Notes for the LLM
- Language: <English | Chinese | "<Primary> (primary) + <Secondary> (secondary)">
                                     # for bilingual, name the PRIMARY first; scaffold also
                                     # writes a "### Bilingual format" block under Naming conventions
- Tone: neutral
```

### Update cadence
- After every ingest: run `build_index.py` (auto-regenerates "Current articles")
- After every lint pass: review open research questions
- Monthly: review scope

## Hierarchical Index Layout

`wiki/index.md` is a **lightweight** anchor file — it should stay small even as
the wiki grows to hundreds of pages. All detailed listings live in per-domain
catalog files under `wiki/index/`.

### wiki/index.md structure (auto-generated)

```markdown
---
type: system-index
graph-excluded: true
updated: YYYY-MM-DD
generated_by: scripts/build_index.py
---

# Index — <Topic>

> Domains: AI, Finance | Sources: 47 | Concepts: 83 | Entities: 31

## Navigation
- [AI domain](index/ai.md) — 38 concepts, 21 sources
- [Finance domain](index/finance.md) — 22 concepts, 14 sources

## Top concepts by domain
### AI
- [[concepts/transformer]] · [[concepts/attention-mechanism]] · ...

### Finance
- [[concepts/dcf-valuation]] · ...

## Recent ingests (last 7 days)
- [[sources/paper-xyz]] — 2026-04-28

## Open Questions
- <first research question>
```

### wiki/index/<domain>.md structure (auto-generated, full detail)

```markdown
---
type: domain-index
domain: ai
graph-excluded: true
updated: YYYY-MM-DD
---

# AI Domain Index

## Concepts (38)
| Page | Confidence | Last Reviewed | Sources |
|------|-----------|---------------|---------|
| [[concepts/transformer]] | high | 2026-04-20 | 8 |
| [[concepts/attention-mechanism]] | medium | 2026-04-10 | 4 |
...

## Sources (21)
| Page | Date | Domain Volatility |
|------|------|-------------------|
| [[sources/attention-paper]] | 2017-06-12 | low |
...

## Entities (12)
...
```

This structure means `query` only needs to load `wiki/index.md` (small) and
then the single relevant domain catalog — not every page.

## YAML Frontmatter — Page Types

### Source Summary (`wiki/sources/<slug>.md`)

```yaml
---
type: source-summary
title: ""
date: YYYY-MM-DD
source_url: ""
domain: ""
author: ""
tags: [wiki, wiki/source]
processed: true
raw_file: ""
raw_sha256: ""
last_verified: YYYY-MM-DD
possibly_outdated: false
language: "English"
canonical_source: ""
---
```

### Personal Writing

```yaml
---
type: personal-writing
title: ""
date: YYYY-MM-DD
status: draft
topic_tags: []
confidence_at_writing: medium
superseded_by: ""
raw_file: ""
raw_sha256: ""
last_verified: YYYY-MM-DD
tags: [wiki, wiki/source]
processed: true
---
```

### Concept (`wiki/concepts/<slug>.md`)

```yaml
---
type: concept
title: ""                    # primary-language name per CLAUDE.md (Chinese-primary vault: "注意力机制（Attention Mechanism）")
date: YYYY-MM-DD
updated: YYYY-MM-DD
tags: [wiki, wiki/concept]
source_count: 0
confidence: low
domain_volatility: medium
last_reviewed: YYYY-MM-DD
aliases: []                  # ALL names — Chinese AND English (+ synonyms)
---
```

### Entity (`wiki/entities/<slug>.md`)

```yaml
---
type: entity
title: ""                    # primary-language name per CLAUDE.md (Chinese-primary vault: "英伟达（NVDA）")
date: YYYY-MM-DD
tags: [wiki, wiki/entity]
entity_type: person          # person | tool | paper | organization
aliases: []                  # ALL names — Chinese AND English (+ ticker/synonyms)
---
```

### Synthesis (`wiki/synthesis/<slug>.md`)

```yaml
---
type: synthesis
title: ""
date: YYYY-MM-DD
tags: [wiki, wiki/synthesis]
source_count: 0
confidence: low
---
```

### Domain Index (`wiki/index/<domain>.md`)

```yaml
---
type: domain-index
domain: ""                   # lowercase, matches a domain in CLAUDE.md (§ Scope)
graph-excluded: true
updated: YYYY-MM-DD
generated_by: scripts/build_index.py
---
```

### System Files

All system files must include `graph-excluded: true`:

```yaml
---
type: system-index        # or system-overview, system-questions, domain-index
graph-excluded: true
updated: YYYY-MM-DD
generated_by: scripts/build_index.py  # for auto-generated files
---
```

Applies to: `wiki/index.md`, `wiki/index/*.md`, `wiki/overview.md`,
`wiki/QUESTIONS.md`, all `outputs/` files.

---

## Forbidden wikilink targets

- System files: `[[log]]` `[[index]]` `[[overview]]` `[[QUESTIONS]]`
- Domain index files: `[[index/ai]]` etc.
- Output files: `[[outputs/...]]`
- Inside `log/`: use plain text paths, never wikilinks
